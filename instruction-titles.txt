칭호 상점 시스템을 추가해줘. 앞 단어 + 뒤 단어 조합형 칭호.

1) 칭호 상점 페이지 (/shop):
- 메인 페이지 로그인 영역에 "칭호 상점" 버튼 추가. 로그인한 사람만.
- 배경 background.png.
- 상단에 내 포인트 표시 + 현재 장착 칭호 표시.
- 탭 2개: "앞 단어" / "뒤 단어"
- 각 탭에서 단어 목록을 카드 그리드로 표시. 모바일 2열, 데스크탑 4열.
- 각 카드: 단어 + 가격 + 구매 버튼.
- 앞 단어, 뒤 단어 각각 1개씩 구매 가능. 둘 다 사면 조합되어 칭호 완성.
- 앞만 사도 되고 뒤만 사도 됨. 둘 다 사면 조합 칭호.
- 이미 다른 사람이 구매한 단어는 "사용 중" 표시, 구매 불가. 선점 방식.
- 다른 단어로 바꾸고 싶으면 새로 구매 (기존 것은 해제되어 다른 사람이 살 수 있게 됨, 포인트 환불 없음).
- 포인트 부족하면 구매 버튼 비활성화.

2) 칭호 규칙:
- 같은 단어를 여러 명이 살 수 없음. 한 단어는 한 사람만 선점.
- 매달 1일 자정에 모든 칭호 리셋 (해제, 포인트 환불 없음).
- 리셋 후 다시 구매 가능.

3) Firestore:
- 컬렉션: "titleWords"
- 각 문서 ID: 단어 ID (예: front_sturdy, back_king)
- 필드: word(string), type(string "front"/"back"), price(number), owner(string 닉네임 또는 빈문자열), purchasedMonth(string "2026-04")
- users/{nickname} 문서에 frontTitle(string), backTitle(string) 필드 추가.
- 매달 리셋 체크: 페이지 로드 시 현재 월과 purchasedMonth 비교, 다르면 owner 초기화.

4) 칭호 표시:
- 닉네임 앞에 「앞단어 뒤단어」 형태로 표시. 앞만 있으면 「앞단어」, 뒤만 있으면 「뒤단어」.
- 댓글, 방명록, 게시판, 채팅, 미니홈피, 흔적 남기기 등 닉네임 나오는 모든 곳에 표시.
- 칭호 색상: 금색 #d4af37.
- 칭호 글자 크기: 닉네임보다 확실히 작게. 닉네임 font-size의 약 70% 정도.
