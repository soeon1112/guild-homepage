정보 게시판 페이지를 만들어줘. Firebase Firestore를 사용해. src/lib/firebase.ts는 이미 있어.

1) Firestore 컬렉션: "board"
각 문서 필드: title(string), content(string), nickname(string), password(string), createdAt(timestamp), updatedAt(timestamp)

2) 게시판 목록 페이지 (/board):
- 배경은 background.png
- 상단에 "정보 게시판" 제목. 파란색 #4a90d9에 흰색 외곽선.
- 글쓰기 버튼 (우측 상단)
- 글 목록: 번호, 제목, 작성자, 날짜 표시
- 목록 배경 rgba(0,0,0,0.4) 반투명, 텍스트 흰색
- 제목 클릭하면 상세 페이지로 이동
- 페이지네이션: 한 페이지에 10개씩, 이전/다음 버튼
- 홈으로 돌아가기 버튼

3) 글쓰기 페이지 (/board/write):
- 닉네임 입력, 비밀번호 입력 (수정/삭제 시 필요), 제목 입력, 본문 textarea
- 등록 버튼, 취소 버튼
- 등록 시 Firestore에 저장하고 목록으로 이동
- 입력 필드 스타일: 배경 rgba(255,255,255,0.1), 흰색 텍스트

4) 상세 페이지 (/board/[id]):
- 제목, 작성자, 날짜, 본문 표시
- 수정 버튼, 삭제 버튼
- 수정 클릭 시 비밀번호 입력 팝업. 비밀번호 맞으면 수정 페이지로 이동.
- 삭제 클릭 시 관리자 비밀번호 "dawnlight2024" 입력해야 삭제 가능.
- 목록으로 버튼

5) 수정 페이지 (/board/edit/[id]):
- 글쓰기와 동일한 폼에 기존 내용 채워져 있음
- 저장 시 updatedAt 갱신

6) 댓글 기능:
- 상세 페이지 하단에 댓글 섹션
- Firestore 서브컬렉션: board/{postId}/comments
- 댓글 필드: nickname(string), content(string), createdAt(timestamp)
- 닉네임 + 댓글 입력 + 등록 버튼
- 댓글 목록은 시간순 표시
- 로그인 불필요

7) 디자인:
- 전체적으로 메인 페이지 분위기와 통일
- 버튼 스타일: 금색 테두리, 금색 또는 파란색 텍스트
- 모바일 퍼스트, 반응형

8) 메인 페이지에서 정보 게시판 버튼 클릭 시 /board로 이동하도록 연결.

이미지에 filter 적용하지 마.
