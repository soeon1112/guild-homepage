메인 페이지에 익명 편지 시스템을 추가해줘.

위치: 로그인 영역과 최신 현황 사이. 감성적으로 꾸며줘.

1) 메인 페이지 편지 UI:
- 왼쪽에 우체통 아이콘, 오른쪽에 종이비행기 아이콘 배치.
- 종이비행기 아이콘은 lucide-react의 Send 아이콘 사용.
- 우체통과 종이비행기 사이에 바람선/궤적 같은 점선 곡선을 CSS로 그어줘.
- 우체통 아래에 "마음을 전해보세요" 같은 감성 문구 작게.
- 텍스트는 파란색 #4a90d9에 흰색 외곽선.
- 로그인한 사람만 우체통 클릭 가능. 안 했으면 "로그인이 필요합니다".
- 우체통 클릭 시 편지 쓰기 모달 열림.
- 새 편지 도착 알림은 종이비행기 아이콘이 금색으로 반짝이는 CSS 애니메이션 (opacity 0.5~1 반복).
- 종이비행기 클릭 시 받은 편지 모달 열림.

2) 편지 쓰기 모달:
- 편지지 느낌의 디자인. 배경 rgba(255,250,240,0.95) 따뜻한 아이보리색.
- "받는 사람" 선택: 길드원 닉네임 드롭다운 목록 (Firestore users 컬렉션에서 가져오기).
- "편지 내용" textarea. 따뜻한 느낌의 폰트.
- "보내기" 버튼. 금색 스타일.
- 보내는 사람은 로그인된 닉네임이 자동으로 들어가지만 화면에 표시하지 않음.
- 보내면 "편지가 우체통에 넣어졌습니다. 곧 전달될 거예요!" 메시지.

3) Firestore 컬렉션: "letters"
필드: from(string 보내는사람닉네임), to(string 받는사람닉네임), content(string), status(string: "pending"/"approved"/"rejected"), createdAt(timestamp), deliveredAt(timestamp), read(boolean)

4) 관리자 페이지 (/admin/letters):
- 관리자 비밀번호 "dawnlight2024" 입력 후 접근.
- 대기중(pending) 편지 목록: 보내는 사람, 받는 사람, 내용 전부 표시.
- 각 편지에 "승인" / "거절" 버튼.
- 승인하면 status를 "approved"로 변경, deliveredAt에 현재 시간 기록.
- 거절하면 status를 "rejected"로 변경.

5) 받은 편지 모달 (종이비행기 클릭 시):
- 편지지 디자인. 따뜻한 아이보리 배경.
- "익명의 누군가로부터" 표시. 보낸 사람 절대 안 보여줌.
- 편지 내용 표시.
- 날짜 표시.
- "읽었습니다" 버튼 누르면 읽음 처리.
- 여러 편지가 있으면 이전/다음으로 넘겨볼 수 있게.

6) 디자인 포인트:
- 우체통 영역 배경: rgba(0,0,0,0.3) 반투명, 둥근 모서리.
- 편지지 모달: 따뜻한 아이보리 배경, 갈색 계열 텍스트, 편지지 테두리 느낌.
- 전체적으로 따뜻하고 감성적인 느낌.

7) 최신 현황에 절대 표시하지 않음. 완전 비밀.

8) 포인트 적립 없음.

이미지에 filter 적용하지 마.
